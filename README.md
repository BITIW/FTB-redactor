# ftbgui

Парсер и графический редактор книг FTB Quests в формате SNBT.

Без аргументов программа сразу открывает графический редактор:

```bash
cargo run
```

Статистику книги можно получить командой `stats`. Принимается как каталог
`ftbquests` целиком, так и непосредственно его подкаталог `quests`:

```bash
cargo run -- stats /path/to/config/ftbquests
```

Сейчас отчёт содержит количество групп глав, глав, квестов, зависимостей,
задач, наград и таблиц наград, распределение задач и наград по типам, битые
зависимости и список крупнейших глав.

В конце выводится техническая статистика: время загрузки и полного выполнения,
число и объём прочитанных SNBT-файлов, скорость обработки, версия сборки и
платформа.

## Интерактивный редактор

Графический редактор на `egui`:

```bash
cargo run -- gui ./my_ftbquests
```

В окне доступны группы и главы, интерактивное полотно с масштабированием и
панорамированием, перетаскивание квестов по координатам FTB, задачи, награды и
зависимости. Двойной клик по полотну создаёт квест. Средняя или правая кнопка
мыши двигает полотно, `Ctrl + колесо` масштабирует, а `Shift` отключает
привязку к сетке при перемещении.

Верхняя панель содержит инструменты «Курсор», «Связи», «Выделение» и
«Удаление» с горячими клавишами `Ctrl+1`…`Ctrl+4`. Рамкой можно выделить
несколько квестов, перемещать их вместе, копировать через `Ctrl+C`, вставлять
через `Ctrl+V` и удалять клавишей `Delete` после подтверждения. Вставленные
квесты получают новые ID, а зависимости внутри скопированной группы
переназначаются на копии. Snap включён по умолчанию; кнопка «К сетке»
выравнивает всё текущее выделение.

Названия групп и глав редактируются прямо в дереве книги. Главу можно
перетащить за маркер `≡` в другую группу или в раздел «Без группы». Кнопка
«Импорт проекта…» открывает системный выбор папки; импортированный проект по
умолчанию экспортируется в соседнюю папку с суффиксом `_edited`, чтобы не
перезаписать оригинал.

Группы глав можно сворачивать по отдельности. Настройки
автосохранения, выбранный инструмент, открытый проект, текущая глава/квест и
положение камеры сохраняются между запусками.

У полей предметов, сущностей и иконок есть справочник с поиском по ID и
человекочитаемому английскому названию. Для каждой книги используется её
собственная папка `<проект>/mods`: туда можно положить `.jar` или распакованные
моды с каталогом `assets`. Для модовых названий используются
`assets/<modid>/lang/en_us.json` и модели предметов; JAR распаковывать не
требуется.

Если item model ссылается на доступную PNG-текстуру, справочник показывает
компактную иконку 24×24, а квест с таким предметом в поле «Иконка» рисует её
внутри фигуры вместо текстового сокращения. Поддерживаются обычные
`layer0`/`layer1`-модели и наследование от block-моделей. Для динамических
моделей и сущностей, которые нельзя достоверно отрисовать без движка
Minecraft, остаётся текстовый fallback.

Для полного vanilla-набора нажмите «Добавить Minecraft JAR…» в справочнике и
выберите клиентский JAR нужной версии. В официальном лаунчере он обычно лежит
в `.minecraft/versions/<версия>/`, а в Prism Launcher — в
`libraries/com/mojang/minecraft/<версия>/`. Редактор проверит JAR, скопирует
его в `<проект>/.ftbgui/minecraft-client.jar` и прочитает из него все
локализованные блоки, предметы и сущности. Служебная копия хранится отдельно
от `mods`, поэтому загрузчик модов не попытается запустить её как мод.

Автосохранение включено по умолчанию и каждые 5 минут обновляет отдельный
снимок `*_autosave`, сохраняя прошлую версию в `*_autosave_previous`. Во
вкладке «Поиск» можно искать по ID, названиям, описаниям, главам, группам,
предметам и сущностям. Вкладка «Логи» содержит ручную и автоматическую
диагностику битых зависимостей и циклов любой длины с маршрутом и подсказкой
по исправлению.

Редактор учитывает расширенные настройки из официальной документации FTB
Quests: подзаголовки и теги, наследуемую форму и масштаб иконки, режимы
видимости, требования зависимостей, optional/repeatable-квесты, cooldown,
последовательное выполнение задач и progression mode. Неизвестные типы задач,
наград и дополнительные SNBT-поля сохраняются без изменений; импортированные
reward tables также переносятся в экспортируемый проект.

При импорте редактор читает `kubejs/lang/*.json` и показывает вместо
конструкций вида `{translation.key}` человекочитаемый текст выбранной локали.
Во вкладке «Языки» можно переключать и добавлять языки. Многоязычный режим
преобразует обычный текст в ключ `ftbgui.<UUID v4>` и сохраняет словари KubeJS
как minified JSON; ключи в SNBT при этом остаются стабильными.

Создание новой книги:

```bash
cargo run -- new ./my_ftbquests
```

Редактор создаёт книгу с одной главой. Через меню можно добавлять квесты,
задачи `checkmark`, `item` и `kill`, награды опытом и предметами, а также
зависимости между квестами. Результат сохраняется в структуру
`my_ftbquests/quests`.
